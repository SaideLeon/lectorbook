Política de Privacidade

LectorBook

Última atualização: 08 de março de 2026

 

1. Introdução

Esta Política de Privacidade descreve como o LectorBook coleta, utiliza e protege as informações dos seus utilizadores. Ao usar o LectorBook, você concorda com as práticas descritas neste documento.

O LectorBook é uma aplicação web desenvolvida como projeto individual para análise de repositórios GitHub com apoio de Inteligência Artificial (IA). O serviço é mantido por um único desenvolvedor independente.

2. Informações que Coletamos

2.1 Informações que você fornece voluntariamente

•       URL de repositórios GitHub que você submete para análise.

•       Chaves de API do Google Gemini carregadas voluntariamente através da interface.

•       Mensagens de texto enviadas no chat com o assistente IA.

•       Gravações de áudio submetidas para transcrição de voz.

2.2 Informações coletadas automaticamente

•       Dados técnicos básicos necessários para o funcionamento do servidor (logs de acesso padrão, endereço IP).

•       Conteúdo de arquivos dos repositórios públicos acessados via API do GitHub.

2.3 O que NÃO coletamos

•       Não criamos nem exigimos contas de utilizador.

•       Não armazenamos histórico de conversas no servidor após o encerramento da sessão.

•       Não recolhemos dados pessoais de identificação como nome, e-mail ou número de telefone.

•       Não utilizamos cookies de rastreamento ou tecnologias de perfil de utilizador.

3. Armazenamento Local

O LectorBook utiliza o IndexedDB do seu navegador para armazenar localmente as chaves de API do Gemini que você carrega. Este armazenamento acontece exclusivamente no seu dispositivo e não é transmitido para os nossos servidores, salvo quando necessário para realizar chamadas às APIs de IA.

Você pode apagar estes dados a qualquer momento limpando os dados do site no seu navegador.

4. Serviços de Terceiros

O LectorBook integra serviços de terceiros para fornecer as suas funcionalidades principais. Ao usar o LectorBook, os seus dados podem ser processados por estas entidades externas, cada uma com as suas próprias políticas de privacidade:

4.1 Google Gemini AI

Utilizado para análise de código, chat com IA, geração de fichas de leitura e síntese de voz (text-to-speech). Os textos e conteúdo de repositórios enviados ao chat são transmitidos à API do Google Gemini para processamento.

•       Política de privacidade: https://policies.google.com/privacy

•       Termos de uso da API: https://ai.google.dev/gemini-api/terms

4.2 Groq (Transcrição de Áudio)

Utilizado para converter gravações de áudio em texto (speech-to-text), através do modelo Whisper. Os ficheiros de áudio gravados são transmitidos à API da Groq para transcrição.

•       Política de privacidade: https://groq.com/privacy-policy/

•       Termos de serviço: https://groq.com/terms-of-use/

4.3 API do GitHub

Utilizada para listar repositórios, obter a estrutura de ficheiros (árvore) e ler o conteúdo de ficheiros em repositórios públicos.

•       Política de privacidade: https://docs.github.com/site-policy/privacy-policies/github-privacy-statement

•       Termos de serviço: https://docs.github.com/site-policy/github-terms/github-terms-of-service

4.4 Google Fonts

Utilizado para carregar as fontes tipográficas Inter e JetBrains Mono. Podem ser registados pedidos de rede para o domínio fonts.googleapis.com.

•       Política de privacidade: https://policies.google.com/privacy

Recomendamos fortemente que consulte as políticas de privacidade de cada um destes prestadores antes de utilizar o LectorBook com dados sensíveis.

5. Segurança das Chaves de API

As chaves de API do Google Gemini são armazenadas localmente no IndexedDB do seu navegador. Embora o LectorBook não as transmita a terceiros além da API do Google Gemini (quando necessário para processamento), a responsabilidade pela segurança das suas chaves é sua. Recomendamos:

•       Gerar chaves com permissões mínimas necessárias.

•       Não partilhar o seu dispositivo com pessoas não autorizadas.

•       Revogar chaves suspeitas diretamente no Google AI Studio.

•       O token do GitHub é configurado exclusivamente no servidor pelo administrador e nunca é exposto ao navegador.

6. Retenção de Dados

O LectorBook mantém um cache temporário no servidor dos dados de repositórios GitHub acedidos, com duração de 12 horas, para reduzir chamadas repetidas à API do GitHub. Após este período, os dados em cache são eliminados automaticamente. Não existe qualquer base de dados persistente de conteúdo de utilizadores.

7. Seus Direitos

Uma vez que o LectorBook não cria perfis de utilizadores nem armazena dados pessoais de forma persistente, não há dados pessoais seus nos nossos servidores para aceder, corrigir ou eliminar. Para exercer direitos relativos a dados processados pelos serviços de terceiros listados na secção 4, contacte diretamente esses prestadores.

8. Menores de Idade

O LectorBook não é direcionado a menores de 13 anos. Não recolhemos intencionalmente informações de crianças. Se tiver conhecimento de que uma criança utilizou o serviço, não existem dados pessoais armazenados que necessitem de eliminação.

9. Alterações a esta Política

Podemos atualizar esta Política de Privacidade periodicamente. A data de última atualização será sempre indicada no início do documento. A utilização continuada do LectorBook após alterações constitui aceitação da política revisada.
